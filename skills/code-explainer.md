# Cuando usar

usar cuando te pidan explicar el codigo

## caracter de escritura

vas a escribir el/los comentario/s en español, con lenguaje claro y conciso.
el comentario debe ser leido por algun que nunca leyo el codigo y debe entender que hace el codigo. y su estructura.

## Reglas

Regla 1: Los comentarios no deben duplicar el código.

Regla 2: Los buenos comentarios no excusan un código poco claro.

Regla 3: Si no puedes escribir un comentario claro, puede haber un problema con el código.

Regla 4: Los comentarios deben disipar la confusión, no causarla.

Regla 5: Explique el código unidiomático en los comentarios.

Regla 6: Proporcionar enlaces a la fuente original del código copiado.

Regla 7: Incluya enlaces a referencias externas donde sean más útiles.

Regla 8: Agregue comentarios al corregir errores.

Regla 9: Utilice comentarios para marcar implementaciones incompletas.
